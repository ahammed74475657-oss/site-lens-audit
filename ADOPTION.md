# Adoption Plan

This project should earn adoption honestly. Do not fake stars, users or testimonials.

## First places to share

- Indie Hackers build log.
- Dev.to post: "I built a tiny local SEO and accessibility audit tool".
- Reddit communities that allow project sharing.
- LinkedIn post with screenshots and a short demo.
- GitHub topic tags: `seo`, `accessibility`, `audit`, `frontend`, `express`.

## First user targets

- Freelance web developers.
- Landing page builders.
- Small agency teams.
- Indie makers shipping marketing pages.
- Students learning technical SEO and accessibility.

## Suggested launch checklist

- Add screenshots to the README.
- Create 5 starter issues.
- Tag `v1.0.0`.
- Publish a GitHub release.
- Pin the repository on your GitHub profile.
- Share a 30-second demo GIF or screenshot thread.

## Honest adoption signals

- Issues from real users.
- Forks with new checks.
- Stars from people who tried it.
- Screenshots of reports generated from real pages.
- Pull requests for new audit rules.
